3. Добавлен файл README с инструкцией и пояснениями.
4. Инициализация БД для пользователя реализована в моем варианте как таблица связей пользователь - слово.
При регистрации нового пользователя, каждый новый пользователь получает 11 системных слов с параметром is_active=True. Для системных слов параметр user_id = None. Когда пользователь изучит слово он может удалить это слово из своего набора слов, в этом случае параметр is_active для его id замениться на False, но системные(общие) слова не удаляются из базы данных. Регистрация нового пользователя и получение 11 системных слов реализованно в функции register_user в файле bd_functions.py.
При созднии слов пользователя они появляются в общей базе слов, но имеют параметр created_by, где прописывается id пользователя, т.е. создается связь. Добавление слова пользователя реализованно в функции add_user_word в файле bd_functions.py. Удаление слова пользователя реализованно в функции delete_user_word в файле bd_functions.py, в этой функции присутствует проверка на наличие слова у пользователя, в функции process_delete_word в файле main.py также присутствует проверка на наличие слова у пользователя и оно выводит сообщение для пользователя что слово не найдено в коллекции пользователя.
